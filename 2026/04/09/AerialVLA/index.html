AerialVLA

[Arxiv-2026]AerialVLA: A Minimalist End-to-End Vision-Language-Action Framework for UAV Navigation

Problems

无人机视觉-语言导航（VLN）方法中存在的两大“拐杖”依赖问题：

  • 密集的“神谕”引导（如每一步都给出“左转”“右转”等精确方向提示），把导航任务降级为被动指令跟随（Fuzzy Directional Prompting）
  • 外部物体检测器（如 Grounding DINO）来触发降落,导致割裂的感知-控制回路（High-DoF Control via Numerical Tokenization）

Innovation

  • 提出首个真正端到端的无人机 VLA 模型，摆脱神谕引导和外部检测器。
  • 设计模糊方向提示 + 数值化动作 token 的轻量架构，提升泛化能力。
  • 在 TravelUAV 上实现 SOTA，尤其在零样本泛化上表现突出。

Method

Minimalist Dual-View Perception

大多数 UAV-VLN 方法（如 TravelUAV、AVDN）使用多相机阵列（通常是 5 个视角：前、后、左、右、下），这种设计的出发点是"信息越全越好"。 但本文指出无人机导航的核心是"向前看 + 向下看"，其他视角是锦上添花，而非必要。 多余的相机可能会造成信息冗余，硬件/计算开销大，过拟合

  • 输入：Ifront∈RH×W×3,Idown∈RH×W×3I_{front} \in \mathbb{R}^{H×W×3}, I_{down} \in \mathbb{R}^{H×W×3}Ifront​∈RH×W×3,Idown​∈RH×W×3
  • 输出：Icomp∈R2H×W×3I_{comp} \in \mathbb{R}^{2H×W×3}Icomp​∈R2H×W×3 Icomp=torch.cat([Ifront,Idown],dim=0)I_{comp} = \text{torch.cat}([I_{front}, I_{down}], dim=0)Icomp​=torch.cat([Ifront​,Idown​],dim=0) Iresized=resize(Icomp,(224,224))I_{resized} = resize(I_{comp}, (224, 224))Iresized​=resize(Icomp​,(224,224))

双编码策略

  • SigLIP：语言对齐的语义特征，理解"目标是什么"
  • DINOv2：细粒度空间表示，理解"目标在哪里"

MLP Projector:将视觉特征映射到LLM的embedding空间

在未见场景双视角比五视角SR提高15%

Fuzzy Directional Prompting

现有方法依赖密集的、逐步骤的oracle指导,这些指导来自预先录制的最优轨迹,让无人机变成了被动的指令跟随者，而非主动的自主导航。

模糊性

  • 空间模糊：只有粗粒度方向，无精确角度
  • 时序模糊：无step-by-step对齐，只有当前时刻的提示
  • 传感器模糊：依赖IMU估算，对齐真实世界的不完美定位

Prompt:[visual token] + [fuzzy directional hint] + [target description]

Geometry-Consistent Supervision:

模糊提示说"目标在你右边" 但专家轨迹在某个时刻是直飞 这可能是：

  • (a) 模型应该右转（矛盾）
  • (b) 前方有障碍物，直飞是避障（合理）

High-DoF Control via Numerical Tokenization

外部物体检测器（如 Grounding DINO）来触发降落,导致割裂的感知-控制回路

Define a continuous 3-DoF action space：A=(Δx,Δy,Δz)\mathcal{A} = (\Delta x, \Delta y, \Delta z)A=(Δx,Δy,Δz)

3-DoF（3自由度）而非6-DoF,将roll/pitch等底层稳定控制委托给飞行控制器,对齐商业无人机API（DJI SDK、PX4）

统一降落策略：训练时：专家轨迹的终止帧被标记为：动作标签：⟨0, 0, 0⟩（零位移）和文本输出：LAND token Standard Numerical Tokenization

RT-2, OpenVL使用特殊token <act_0>~<act_255>，需要从零学习 embedding + 序数关系， UAV数据量小,冷启动问题严重

将连续动作离散化为 99个bin，直接接映射到LLM已有的数值token（如"0", "1", ..., "98"），无需引入新的特殊token，利用预训练模型的数值理解能力（知道0<1<2），更快的收敛速度，更平滑的控制轨迹
